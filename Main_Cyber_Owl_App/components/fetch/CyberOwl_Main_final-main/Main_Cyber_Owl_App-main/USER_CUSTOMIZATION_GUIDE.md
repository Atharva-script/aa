# User Customization & Data Management Guide

This guide explains how to manually customize the Cyber Owl application's behavior, appearance, and settings by editing specific raw data files.

## 1. Detection Filters (Blocklists)
These are the most important "raw data" files for the application's core functionality. They define the vocabulary used for abuse and profanity detection.

- **Location**: `csv/` folder
- **Files**:
  - `english.csv` (Primary English blocklist)
  - `hindi.csv`, `chinese.csv` (Multi-language support)
  - `whisper_transcripts.csv` (Used for audio transcription monitoring)

**Action**: You can **add new words** to these CSV files to block specific terms or **remove words** to prevent false positives.

> **Solution**: "If the app is missing a bad word or blocking a safe one, open the relevant `.csv` file and simply add or delete that word."

## 2. Email Notification Templates
These HTML files control the design and content of the emails sent by the system (e.g., when monitoring stops or a threat is detected).

- **Location**: `email_system/templates/`
- **Files**:
  - `stop_monitoring.html` (Notification when monitoring stops)
  - `abuse_content_detected.html` (Abuse alert)
  - `nude_content_detected.html` (Nudity alert)
  - ...and others.

**Action**: You can edit the HTML/CSS to change the **colors (branding)**, **logo**, or **text messages** (e.g., "The Cyber Owl protection system has been paused").

> **Solution**: "To change how email alerts look (e.g., add your company logo or change the message), edit these HTML files."

## 3. Configuration & Credentials
This controls the "plumbing" of the application (email server, API keys).

- **Location**: Root directory
- **Files**:
  - `.env` (or `.env.template`)

**Action**: Update variables like `GMAIL_APP_PASSWORD`, `EMAIL_SENDER`, etc.

> **Solution**: "If emails aren't sending, you manipulate this file to update the credentials."

## 4. Logs & History (Output Data)
These are data files generated by the app. While you don't usually *edit* them to change behavior, you can read them to understand what happened or delete them to reset history.

- **Location**: Root directory
- **Files**:
  - `screen_monitor.log` (Log of screen activity/errors)
  - `backend_startup.log` (Startup diagnostics)
  - `detection_logs.json` (History of detected threats)
  - `users.db` (SQLite database containing user accounts)

> **Solution**: "To debug an error or clear your detection history, you would check or delete these files."

---

## Summary of Actions

| File Type | Location | What you can do |
| :--- | :--- | :--- |
| **Logic/Filters** | `csv/*.csv` | **Add/Delete words** to customize what the AI detects as "abuse". |
| **Visuals** | `email_system/templates/*.html` | **Rebrand emails** with your own text, colors, and logos. |
| **Settings** | `.env` | **Fix connection issues** by updating passwords/API keys. |
